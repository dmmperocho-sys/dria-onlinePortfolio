Godzilla Sumobot

ESP32  ·  C++ / Arduino  ·  3D Printing

Introduction

Godzilla is a dual-MCU autonomous sumo robot built for competitive robot sumo events. It combines advanced sensing, voice feedback, and a reinforced scraper chassis into a single aggressive package. Designed to push, outmaneuver, and dominate the ring — Godzilla isn't just a robot, it's a statement.

Basic features
↑ Move forward
↓ Move backward
← Turn left
→ Turn right
Unique features
Turn signals
Godzilla uses built-in LEDs as directional indicators. Each turn triggers the corresponding LED to blink in sync with the robot's steering — a practical visual cue during manual control and a nod to real vehicle behavior.
Honk
A dedicated honk command activates the buzzer and plays an audio clip through the DFPlayer Mini and speaker. Godzilla literally roars — whether it's intimidating opponents or signaling readiness before a match, the honk is as iconic as the robot itself.
Operating modes
Hockey mode
Godzilla swaps the sumo ring for a makeshift rink. It tracks and strikes a target puck-like object using its reinforced scraper, combining speed and precision for a different kind of robot sport.
Sumobot mode
The core competitive mode. Godzilla uses its sensors to detect opponents and the ring boundary, then autonomously charges, pushes, and outmaneuvers rivals to force them out — full aggression, no mercy.
Maze solver
Godzilla shifts from brute force to brains. Using its onboard sensors, it navigates a maze by detecting walls and open paths, finding its way to the exit through logic and real-time decision making.
GUI controls

Godzilla includes a graphical user interface for manual control — allowing the operator to switch between operating modes, trigger the honk, and send directional commands in real time.
